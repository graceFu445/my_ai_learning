# 智能FAQ检索系统 - 项目结构说明文档

## 项目概述

本项目是一个基于 FastAPI、LlamaIndex、FAISS 和 DashScope 的智能 FAQ 检索系统 MVP。系统采用 RAG（Retrieval-Augmented Generation）架构，支持 FAQ 条目管理、CSV 批量导入、向量检索、多轮会话记录和受控回答生成。

MVP 阶段重点是后端能力闭环：FAQ 原文和会话历史使用 JSON 文件保存，LlamaIndex 负责编排文档、索引和 retriever，FAISS 负责本地向量相似度检索，DashScope 用于文本向量化和自然语言回答生成。

## 技术架构

### 核心技术栈

- **API框架**：FastAPI - 提供 RESTful API 和 Swagger 调试页面
- **Web服务器**：Uvicorn - ASGI 应用服务器
- **AI框架**：LlamaIndex - 组织 FAQ 文档、向量索引和 retriever 检索流程
- **向量数据库**：FAISS - 本地高性能相似度检索
- **模型服务**：DashScope SDK - 提供 Embedding 和回答生成
- **数据存储**：JSON 文件 - MVP 阶段保存 FAQ 原文和会话历史
- **环境管理**：Conda - 项目必须使用 `faq-rag` 独立环境运行
- **测试框架**：pytest - 覆盖存储层、服务层和 API 层

### 系统架构图

```text
用户请求
  ↓
FastAPI 路由层
  ↓
业务服务层（FAQService / ChatService）
  ↓
JSON 存储层 ←→ LlamaIndex + DashScope + FAISS 检索层
  ↓
DashScope 受控回答生成
  ↓
响应返回
```

## 核心模块详解

### 1. main.py - FastAPI Web服务入口

**功能**：创建 FastAPI 应用，组装存储、索引、回答生成器和业务服务，并暴露 HTTP 接口。

**主要特性**：

- 健康检查接口
- FAQ 增删改查接口
- CSV 批量导入接口
- 聊天问答接口
- 会话历史查询接口
- 手动重建索引接口
- 测试时支持注入临时数据目录、假索引和假回答生成器

**核心API端点**：

```text
GET    /health                  # 健康检查
POST   /faqs                    # 创建FAQ
GET    /faqs                    # 查询FAQ列表
GET    /faqs/{faq_id}           # 查询单条FAQ
PUT    /faqs/{faq_id}           # 更新FAQ
DELETE /faqs/{faq_id}           # 删除FAQ
POST   /faqs/import             # CSV批量导入FAQ
POST   /chat                    # 用户问答
GET    /sessions/{session_id}   # 查询会话历史
POST   /index/rebuild           # 手动重建索引
```

### 2. config.py - 配置管理模块

**功能**：集中读取 `.env` 和环境变量，避免业务代码直接依赖系统环境。

**主要配置**：

```python
DASHSCOPE_API_KEY               # DashScope API密钥
DASHSCOPE_CHAT_MODEL            # 聊天模型名称，默认 qwen-plus
DASHSCOPE_EMBEDDING_MODEL       # 向量模型名称，默认 text-embedding-v4
EMBEDDING_DIMENSION             # 向量维度，text-embedding-v4 默认使用 1024
TOP_K                           # 检索返回数量，默认3
MIN_SIMILARITY_SCORE            # 最低相似度阈值，默认0.0
```

### 3. schemas.py - API请求/响应定义

**功能**：定义接口请求和响应 Schema，用于参数校验和 Swagger 文档展示。

**主要模型**：

- `FaqCreate`：创建 FAQ 请求
- `FaqUpdate`：更新 FAQ 请求
- `ChatRequest`：聊天请求
- `ChatResponse`：聊天响应
- `ImportResponse`：CSV 导入响应

### 4. json_store.py - JSON数据存储

**功能**：负责 FAQ 原文和会话历史的本地 JSON 持久化。

**主要特性**：

- 自动创建 `data/faqs.json`
- 自动创建 `data/sessions.json`
- FAQ 条目带唯一 ID 和时间戳
- 会话历史按 `session_id` 追加消息
- JSON 使用 `ensure_ascii=False`，方便直接阅读中文内容

### 5. faq_service.py - FAQ业务服务

**功能**：封装 FAQ 管理业务逻辑，并在数据变化后触发索引重建。

**主要特性**：

- FAQ 列表查询
- FAQ 单条查询
- FAQ 创建、更新、删除
- CSV 批量导入
- 标签字段清洗
- 自动重建 FAISS 索引

### 6. chat_service.py - 聊天业务服务

**功能**：处理一次用户提问的完整业务流程。

**主要特性**：

- `session_id` 为空时自动创建新会话
- 保存用户问题和助手回答
- 读取最近几轮会话历史
- 调用向量索引检索相关 FAQ
- 根据相似度阈值过滤结果
- 调用回答生成器输出受控回答

### 7. index.py - LlamaIndex + DashScope + FAISS索引模块

**功能**：将 FAQ 转换为 LlamaIndex Document，调用 DashScope 生成 embedding，构建和加载 FAISS 索引，并通过 LlamaIndex retriever 执行语义检索。

**主要特性**：

- FAQ 文档化
- LlamaIndex 编排索引构建、持久化和 retriever 检索
- DashScope Embedding 向量化
- FAISS 本地索引持久化
- 服务重启后自动加载已有索引
- 检索结果映射回原始 FAQ JSON
- 缺少 DashScope 配置时使用 `EmptyIndex`，保证基础 CRUD 可演示

### 8. answer.py - DashScope回答生成模块

**功能**：将检索命中的 FAQ 和会话历史组织为提示词，调用 DashScope 的 Qwen 模型生成回答。

**主要特性**：

- 只允许基于命中的 FAQ 内容回答
- 保留最近会话历史用于追问理解
- 缺少 DashScope 配置时使用占位生成器，返回清晰错误

## 工作流程说明

### 1. 服务启动流程

```text
读取 .env 配置
  ↓
初始化 JSON 存储
  ↓
根据 DashScope 配置创建 FAISS 索引或空索引
  ↓
根据 DashScope 配置创建回答生成器或占位生成器
  ↓
组装 FAQService 和 ChatService
  ↓
启动 FastAPI 服务
```

### 2. FAQ导入流程

```text
上传 CSV 文件
  ↓
解析 question、answer、tags 字段
  ↓
写入 data/faqs.json
  ↓
将 FAQ 转换为 LlamaIndex Document
  ↓
  重建 FAISS 索引
  ↓
返回导入数量和FAQ列表
```

### 3. 问答查询流程

```text
用户提交问题
  ↓
获取或创建 session_id
  ↓
保存用户问题到会话历史
  ↓
FAISS 检索相关 FAQ
  ↓
过滤低相似度结果
  ↓
DashScope 基于命中 FAQ 生成受控回答
  ↓
保存助手回答到会话历史
  ↓
返回 answer、session_id、matched_faqs
```

### 4. 多轮会话流程

```text
第一次请求不传 session_id
  ↓
后端自动创建 session_id
  ↓
后续请求带同一个 session_id
  ↓
后端读取最近几轮历史作为上下文
  ↓
形成同一会话中的连续问答
```

## 部署和使用指南

### 1. 创建Conda环境

```bash
conda env create -f environment.yml
conda activate faq-rag
```

### 2. 配置环境变量

```bash
cp .env.example .env
```

然后编辑 `.env`：

```bash
DASHSCOPE_API_KEY=你的DashScope API Key
DASHSCOPE_CHAT_MODEL=qwen-plus
DASHSCOPE_EMBEDDING_MODEL=text-embedding-v4
EMBEDDING_DIMENSION=1024
TOP_K=3
MIN_SIMILARITY_SCORE=0.0
```

### 3. 启动Web服务

```bash
conda run -n faq-rag uvicorn app.main:app --host 127.0.0.1 --port 8000
```

访问 Swagger 调试页面：

```text
http://127.0.0.1:8000/docs
```

### 4. 运行测试

```bash
conda run -n faq-rag pytest tests -q
```

## 数据格式说明

### CSV导入格式

```csv
question,answer,tags
如何退货？,签收后7天内可申请退货。,售后;退货
配送多久？,普通地区1到3天送达。,配送;时效
```

`tags` 支持用英文分号、中文分号、英文逗号、中文逗号分隔。

### 聊天请求格式

```json
{
  "question": "我想退货怎么办？",
  "session_id": "可选；为空时自动创建"
}
```

### 聊天响应格式

```json
{
  "answer": "系统回答",
  "session_id": "会话ID",
  "matched_faqs": [
    {
      "id": "FAQ ID",
      "question": "命中的FAQ问题",
      "answer": "命中的FAQ答案",
      "tags": ["标签"],
      "score": 0.92
    }
  ]
}
```
